# UI Automation Project: XYZ Bank

---
## 📝 Описание
Проект для автоматизации тестирования раздела "Manager" на сайте
https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager
Реализован с использованием Selenium WebDriver, Pytest и Allure Reports по паттерну Page Object Model. 

## 🚀 Технологии 
- Python 3.10+
- Selenium WebDriver
- Pytest
- Allure Framework
- WebDriver Manager

## ⚙️ Установка и запуск

### 1️⃣ Клонирование проекта и установка зависимостей

`git clone https://github.com/yourusername/sdet_praktikum.git
cd sdet_praktikum
python -m venv .venv` 
## Windows
`.venv\Scripts\activate`
## Установите зависимости
`pip install -r requirements.tx`

### 2️⃣ Настройка окружения

- Создайте файл .env на основе .env.example.
- На Windows это можно сделать командой:

`copy .env.example .env`

### 3️⃣ Запуск тестов

`pytest -vv`

### 4️⃣ Формирование отчёта Allure

`pytest -vv --alluredir=allure-results` 

`allure serve allure results`

## 🔄 Отчёты и CI/CD

Проект легко интегрируется с GitHub Actions для автоматического запуска тестов и публикации отчетов.

- Триггеры запуска: Workflow запускается автоматически при каждом push Pull Request в ветку master.
- Основная задача: Пайплайн выполняет полный прогон всех UI-тестов в headless - режиме на чистом окружении.


## ✅ Позитивный тест-кейс №1

---
ID: ТС001
Название: Успешное добавление клиента

Предусловие:
Открыта страница [Bank Manager Login → Add Customer](https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager/addCust)

Шаги:
1. Перейти во вкладку Add Customer.  
2. Сгенерировать тестовые данные:  
   - Post Code — 10-значный номер  
   - First Name — имя на основе Post Code  
   - Last Name — “AutoTest”  
3. Заполнить поля формы данными.  
4. Нажать кнопку Add Customer.  
5. Подтвердить системный alert.  
6. Перейти на вкладку Customers.  
7. Проверить, что созданный клиент отображается в таблице.

Ожидаемый результат:
Клиент успешно добавлен, присутствует в таблице клиентов.

## ✅ Позитивный тест-кейс №2
ID: ТС002
Название: Проверка сортировки клиентов по имени (first name)

Предусловие:
В системе есть не менее 3 клиентов с различными именами.

Шаги:
1. Перейти во вкладку Customers.  
2. Кликнуть по заголовку First Name для сортировки.  
3. Считать текущий порядок отображения имён.  
4. Проверить, что список имен отсортирован (по возрастанию или убыванию).  

Ожидаемый результат:
Имена клиентов отображаются в отсортированном порядке без ошибок.

## ✅ Позитивный тест-кейс №3
ID: ТС003
Название: Успешное удаление клиента

Предусловие:
В системе есть не менее 3 клиентов с различными именами.

Шаги:
1. Перейти во вкладку Customers.  
2. Получить из таблицы Customers список имён.  
3. Узнать длину каждого имени.  
4. Найти среднее арифметическое получившихся длин.
5. Найти клиента, у которого длина ближе к среднему арифметическому.
6. Удалить этого клиента.

Ожидаемый результат:
Клиент, чьё имя было ближе к среднему значению удалён.
